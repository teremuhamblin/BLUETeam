# 🛡️ Procédures SOC‑N5  
### *Actions tactiques, techniques et opérationnelles*

Ce document décrit les **procédures standardisées** utilisées par le SOC‑N5 pour répondre aux incidents.

---

# 🔹 1. Procédure de triage initial (N1)

1. Identifier l’alerte  
2. Vérifier la source (SIEM, EDR, IDS)  
3. Collecter les métadonnées  
4. Classifier la criticité  
5. Escalader si nécessaire  

---

# 🔹 2. Procédure d’analyse (N2)

1. Récupération des artefacts  
2. Analyse des logs  
3. Extraction des IOC  
4. Mapping MITRE  
5. Hypothèses de compromission  
6. Détection de persistance  

---

# 🔹 3. Procédure de réponse (N3)

1. Containment (isolation réseau)  
2. Neutralisation (kill process, remove persistence)  
3. Remédiation (patch, durcissement)  
4. Validation post‑incident  
5. Rapport final  

---

# 🔹 4. Procédure de hunting

1. Définir une hypothèse  
2. Identifier les sources de logs  
3. Construire les requêtes  
4. Corréler les événements  
5. Documenter les résultats  

---

Fin de transmission.
